Processor

Wat is een proceesor?

Een processor, ook wel bekend als CPU (Engels: central processing unit), is een stuk hardware in een computer die instaat voor basisbewerkingen en -controle bij het uitvoeren van programmacode. De centrale verwerkingseenheid (CPU) van een apparaat werkt als zijn hersenen en vertelt andere onderdelen wat ze moeten doen. Een processor zit in iedere computer, smartphone, rekenmachine, tv-bakje, noem maar op. Het is onmisbaar in je computer, zonder processor start hij zelfs niet op. De processor is verantwoordelijk voor het ‘sturen’ van je computer; het verwerkt data, speelt die data door aan andere key components zoals je grafische kaart, RAM-geheugen, SSD/HDD en meer. Daarbovenop is het verantwoordelijk voor alle berekeningen die je computer maakt.

Wat doet een processor?

De aansluitingen van een processor bestaan in hoofdzaak uit een stel besturings-, adres- en datalijnen die aangesloten zitten op de besturingsbus, de adresbus en de databus van de computer. Via de adresbus geeft de processor aan op welk adres van het extern geheugen er iets met de data moet gebeuren, via de databus worden de data getransporteerd, en via de besturingsbus wordt aangegeven of de data gelezen of geschreven moeten worden. De processor communiceert met de buitenwereld doordat bepaalde geheugenadressen met interfaces van randapparatuur zijn verbonden, zoals toetsenborden, beeldschermen, etc. Een moderne CPU bestaat vaak uit meerdere kernen. Elke kern kan onafhankelijk taken uitvoeren, waardoor de CPU meerdere processen tegelijk kan afhandelen. Dit wordt ook wel multitasking genoemd.
De taken van de processor kan je verdelen in fases:

De ophaalfase
De processor haalt eerst de data op die nodig is om het programma te laten werken. Deze data staat opgeslagen in het RAM-geheugen. In de programmateller van de computer staat een adres van het geheugen waarop de nieuwe uit te voeren opdracht staat. In deze fase wordt deze instructie opgehaald.

De decodeerfase
In deze fase wordt er bepaald wat de processor met de opdracht moet doen. De processor past zich aan en gaat zijn eigen processen in de juiste volgorde zetten om de ‘opdracht’ te kunnen uitvoeren.

De uitvoerfase
Alle processen staan goed, de berekening wordt opgestart en de opdracht wordt uitgevoerd.

De opslagfase
In deze fase wordt het resultaat opgeslagen. Het resultaat wordt in een tijdelijk geheugenblok geplaatst.
